# Free Mentors

A full-stack mentorship platform that connects users with experienced mentors for free, one-on-one guidance sessions. Users browse mentors, request sessions, and leave reviews; mentors accept or reject requests; admins manage the community.

## Stack

| Layer    | Tech                                                            |
| -------- | --------------------------------------------------------------- |
| Backend  | Python 3.11, Django 5, Graphene-Django (GraphQL), JWT auth      |
| Database | SQLite by default (Djongo + MongoDB when `MONGO_URI` is set)    |
| Frontend | React 18, TypeScript, Vite, MUI v6, Zustand, TanStack Query     |
| Tests    | pytest (backend), Vitest + React Testing Library (frontend)     |
| Infra    | Docker + docker-compose (backend image + nginx-served frontend) |

The frontend is dark-only with a military-green primary palette. The backend exposes a single `/graphql/` endpoint with GraphiQL enabled.

## Quick start (local)

### Prerequisites
- Python 3.11+
- Node.js 20+
- (optional) Docker 24+ and docker-compose

### Backend

```bash
cd freementors
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\Activate.ps1
pip install -r requirements.txt
python manage.py migrate --run-syncdb
python manage.py seed_db           # creates an admin, mentors, and demo users
python manage.py runserver 0.0.0.0:8000
```

GraphQL playground: <http://localhost:8000/graphql/>

### Frontend

```bash
cd frontend
npm install
npm run dev -- --host 0.0.0.0 --port 5173
```

App: <http://localhost:5173>

---

## Run with Docker

```bash
docker compose up --build
```

This starts:
- `backend` on port **8000** (Django runserver, seeds the DB on first start)
- `frontend` on port **5173** (built static bundle served by nginx with SPA fallback)

The frontend image is a multi-stage build (Node 20 → nginx 1.27-alpine).

To stop and remove containers:
```bash
docker compose down
```

## Testing

### Backend
```bash
cd freementors
pytest
```

### Frontend
```bash
cd frontend
npm test                # one-shot run
npm run test:watch      # interactive
npm run test:coverage   # with coverage report (HTML + lcov in coverage/)
```

Frontend coverage thresholds enforced by Vitest:

| Metric     | Threshold |
| ---------- | --------- |
| Lines      | 60%       |
| Statements | 60%       |
| Branches   | 50%       |
| Functions  | 40%       |

Current run reports **65% line coverage** across 17 test files (61 tests). Files generated by tools (shadcn UI primitives in `src/components/ui`, type-only files, and the GraphQL query strings in `src/lib/queries.ts` / `src/lib/mutations.ts`) are excluded from coverage.

## GraphQL highlights

- `register`, `login` — user registration and JWT login
- `allMentors`, `allUsers` (admin only)
- `createSession`, `updateSession` — request and accept/reject mentorship sessions
- `mySessions` — sessions for the current user (mentee or mentor view)
- `createReview`, `allReviews` — leave and list reviews
- `toggleMentor` (admin) — promote or demote a user
- `hideReview` (admin) — moderate inappropriate reviews

The frontend sends a `Bearer` token from `localStorage["free_mentors_token"]` on every GraphQL request via the axios client in `src/lib/graphql.ts`.

## Notes

- Authentication is persisted in `localStorage` so a refresh keeps you signed in.
- Routes are guarded by `<ProtectedRoute>` and `<RedirectIfAuthed>`; role-protected routes (e.g. `/dashboard/admin`) redirect home if the role does not match.
- The MUI theme is dark-only and lives in `src/lib/muiTheme.ts`. Primary HSL is `127 19% 28%` (military green).
